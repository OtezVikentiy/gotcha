# Политика безопасности

## Сообщение об уязвимости

Пожалуйста, **не** открывайте публичную задачу (issue) на GitHub/GitFlic по
уязвимостям безопасности. Вместо этого сообщите приватно одним из способов:

- Откройте приватный security advisory на хостинге репозитория (если платформа
  хостинга, на которой вы это читаете, поддерживает приватные отчёты об
  уязвимостях, например GitHub Security Advisories), либо
- Напишите мейнтейнерам на: otezvikentiy@gmail.com

Пожалуйста, включите:

- Описание уязвимости и её потенциального воздействия.
- Шаги воспроизведения (минимальный repro очень помогает).
- Затронутую версию/коммит.

Мы подтвердим получение отчёта как можно скорее и будем работать с вами над
исправлением и графиком раскрытия. Пожалуйста, дайте нам разумное время на
устранение проблемы до любого публичного раскрытия.

## Поддерживаемые версии

У Gotcha пока нет пронумерованных релизов; исправления безопасности применяются
к ветке `main`. Как только появятся релизы с тегами, этот раздел будет обновлён,
чтобы указать, какие линейки релизов получают исправления безопасности.

## Позиция по безопасности

Несколько значений по умолчанию, о которых стоит знать при эксплуатации Gotcha:

- **Очистка приватности включена по умолчанию.** IP-адреса и email-адреса
  сообщающего обнуляются на сервере (`GOTCHA_SCRUB_IP`, `GOTCHA_SCRUB_EMAIL`,
  оба по умолчанию `true`), а denylist чувствительных имён ключей (пароли,
  токены, cookie, API-ключи и т. д.) по умолчанию редактируется из
  тегов/контекстов/стек-трейсов/данных спанов (`GOTCHA_SCRUB_KEYS`).
- **Защита от SSRF включена по умолчанию.** Исходящие запросы, выполняемые от
  вашего имени — аптайм-проверки и доставка вебхук-оповещений — отказываются
  обращаться к приватным/loopback/link-local адресам, если вы явно не включите
  это через `GOTCHA_SSRF_ALLOW_PRIVATE=true`. Оставляйте это выключенным на
  любом мультитенантном или доступном из интернета инстансе.
- **Значение `GOTCHA_SECRET_KEY` по умолчанию публично** (оно поставляется в
  исходниках), и процесс отказывается стартовать в режиме `web`/`all` при
  не-локальном `GOTCHA_BASE_URL`, пока не настроен реальный секрет — см. раздел
  «Конфигурация» в README.

Если вы найдёте пробел в любом из вышеперечисленного, пожалуйста, сообщите о нём
так, как описано выше, а не открывайте публичную задачу.
