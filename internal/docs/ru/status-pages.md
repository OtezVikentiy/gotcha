# Публичные статус-страницы

Публичная статус-страница — витрина состояния выбранных [мониторов](/docs/uptime) проекта: отдельная страница без входа в систему, на которую можно давать ссылку пользователям и партнёрам. Настраивается на `/projects/{id}/statuspages` (доступно владельцу/админу организации проекта).

## Создание статус-страницы

1. Откройте боковую панель раздела «Uptime» → «Status Pages».
2. Нажмите «New status page» — откроется модальная форма создания.
3. Заполните поля:
   - **Slug** — короткий идентификатор в URL страницы (например, `main`). Тот же формат, что и у слагов проектов/команд; должен быть уникален в рамках инсталляции.
   - **Title** — заголовок, который увидят посетители.
   - **Description** — короткое описание под заголовком (необязательно).
   - **Enabled** — включает страницу; выключенная страница по-прежнему видна в кабинете, но недоступна публично.
4. В списке мониторов отметьте чекбоксом те, что должны попасть на страницу, и для каждого задайте **Display name** — публичное имя. Оно не обязано совпадать с внутренним именем монитора: настоящее имя видит только администратор в форме, наружу уходит только Display name. Порядок отметки определяет порядок плиток на странице.
5. Сохраните — «Create».

Позже страницу можно отредактировать той же формой или удалить кнопкой «Delete». У сохранённой страницы под формой показывается готовая публичная ссылка.

## Публичный URL

Статус-страница доступна по адресу:

```
{base_url}/status/{slug}
```

Ссылку можно найти прямо под формой редактирования на `/projects/{id}/statuspages` — её удобно скопировать и разместить, например, в футере вашего сайта или в канале поддержки.

## Что видит посетитель

Публичная страница (`/status/{slug}`) не требует входа и не содержит навигации кабинета — только:

- общий статус («Все системы работают» / «Частичный сбой» / «Крупный сбой»);
- плитки сервисов с их Display name, текущим статусом (работает/недоступен/пауза/обслуживание/неизвестно) и полоской доступности, плюс % доступности за 90 дней;
- ленту инцидентов за последние 90 дней (сервис, начало, длительность — без деталей причины и регионов);
- список ближайших запланированных [окон обслуживания](/docs/maintenance).

Реальные URL, хосты, порты мониторов и текст ошибок на публичную страницу не попадают — наружу видно только то, что вы явно выбрали и подписали.
