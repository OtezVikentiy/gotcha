# Термины

Короткий словарь понятий Gotcha — от организаций и проектов до профилей и квот. Там, где у термина есть отдельная страница с подробностями, дана ссылка.

**Организация (organization)** — верхний уровень: объединяет проекты, участников (с ролями) и общие настройки (лимиты приёма, SSO, шаблоны каналов оповещений). Пользователь может состоять в нескольких организациях. Создаётся вместе с первым проектом на `/onboarding`.

**Проект (project)** — отдельное приложение или сервис внутри организации. У каждого проекта свой набор DSN-ключей, свои проблемы, транзакции, метрики, профили и настройки (семплирование, пороги регрессий, ретенция). События из двух разных проектов никогда не смешиваются.

**Команда (team)** — группа участников организации с общим доступом к подмножеству проектов. Используется, чтобы не выдавать права «на всю организацию» каждому, кому нужен доступ лишь к паре проектов.

**DSN** — строка подключения проекта вида `https://<public_key>@<адрес_gotcha>/<id_проекта>`, которую вы передаёте в SDK при инициализации. Содержит публичный ключ (по нему Gotcha определяет проект и организацию) и адрес приёма событий; секретной части, в отличие от старого формата Sentry DSN, нет. Найти и перевыпустить DSN можно в «Настройках проекта» или на странице «Подключение» — см. [SDK и интеграции](/docs/sdk).

**Событие (event)** — один конкретный случай: ошибка со стектрейсом, уровнем, тегами, контекстом, данными пользователя и SDK, присланная в Gotcha по протоколу приёма Sentry. Несколько событий с одинаковым фингерпринтом схлопываются в одну проблему.

**Проблема (issue)** — группа одинаковых событий, сведённых по фингерпринту: одна карточка вместо потока дублей. У проблемы есть статус (не решено/решено/игнорируется), ответственный и счётчик количества событий. Подробнее: [Проблемы](/docs/issues).

**Группировка / фингерпринт (fingerprint)** — ключ группировки событий в проблему, вычисляемый по типу ошибки, сообщению и месту в коде (стеку вызовов). Новое событие с уже встречавшимся фингерпринтом увеличивает счётчик существующей проблемы; с новым — открывает новую проблему (или переоткрывает решённую — это и есть регрессия).

**Транзакция (transaction)** — единица производительности: один обработанный запрос или операция целиком, с общей длительностью и статусом. Отправляется SDK, если включён трейсинг (`traces_sample_rate`/`tracesSampleRate` > 0). Подробнее: [Производительность](/docs/performance).

**Трейс (trace)** — сквозная цепочка связанных транзакций и спанов одного запроса, в том числе через границы сервисов (если `trace_id` прокидывается между ними). Из детали проблемы можно перейти к связанному трейсу, если ошибка произошла внутри отслеживаемого запроса.

**Спан (span)** — вложенный отрезок внутри транзакции: запрос к БД, HTTP-вызов к другому сервису, рендер шаблона и т. п. Транзакция — это, по сути, корневой спан плюс дерево дочерних.

**Web Vitals** — метрики скорости страницы, измеренные в браузере реального пользователя: LCP, INP, CLS, FCP, TTFB. Показываются с перцентилем p75 и рейтингом good/needs-improvement/poor. Собираются браузерным SDK автоматически при включённом трейсинге. Подробнее: вкладка Web Vitals в [Производительности](/docs/performance).

**Метрика (metric)** — числовой временной ряд (counter/gauge/histogram) с произвольными лейблами (окружение, релиз, регион и т. п.), присланный по протоколу OTLP. Для просмотра доступны агрегации p50/p95/avg/sum за период. Подробнее: [Метрики](/docs/metrics).

**Профиль (profile) / флеймграф (flame graph)** — снимок того, куда уходит процессорное время или время выполнения внутри приложения во время реального запроса, собранный SDK (Sentry-профилирование) или загруженный напрямую в формате pprof. Флеймграф визуализирует профиль как стек вызовов: ширина блока — доля времени, вложенность — глубина вызова. Подробнее: [Профилирование](/docs/profiling).

**Монитор (uptime monitor)** — периодическая проверка доступности эндпойнта (HTTP/TCP/DNS/heartbeat) по расписанию. При нескольких подряд неудачных проверках открывается инцидент. Подробнее: [Аптайм](/docs/uptime).

**Инцидент (incident)** — период, когда монитор или показатель регрессии считается «упавшим»: имеет начало, (опционально) конец и историю проверок/событий внутри. По инциденту может сработать оповещение и обновиться публичная статус-страница.

**Канал оповещения (alert channel)** — способ доставки уведомления: email, webhook или Telegram-бот. Настраивается один раз в проекте и переиспользуется в нескольких правилах.

**Правило оповещения (alert rule)** — условие, при котором нужно сработать: новая проблема, регрессия (проблема снова открылась), всплеск ошибок за окно времени или превышение порога по метрике. Привязывается к одному или нескольким каналам доставки; срабатывания одного правила троттлятся, чтобы не заваливать команду дублями. Подробнее: [Оповещения](/docs/alerts).

**Регрессия (regression)** — статистически значимое ухудшение показателя (латентности эндпойнта, Web Vitals, времени в профиле) относительно базовой линии за предыдущий период, обнаруженное автоматически. Регрессии производительности показываются как проблемы в [Производительности](/docs/performance).

**Ретенция (retention)** — срок, в течение которого данные конкретного вида (события/транзакции, спаны трейсов, метрики, профили) хранятся в ClickHouse, прежде чем удаляются. Настраивается по умолчанию для инстанса и может быть сужена (но не расширена) на уровне проекта в «Настройках проекта».

**Квота (quota)** — месячный лимit на приём определённого вида данных (события, транзакции, метрики, профили) на организацию. При исчерпании квоты новые единицы этого вида отклоняются (ingest отвечает `429`) до начала следующего месяца; уже принятые данные не удаляются. Настраивается в «Настройках организации → Использование и лимиты»; `0` означает «без ограничений».

**Окружение (environment)** — произвольная строка (`production`, `staging`, `dev`…), которую SDK прикладывает к каждому событию, транзакции и метрике. Используется для фильтрации почти во всех разделах — так продовые ошибки не тонут среди локальной разработки.

**Релиз (release)** — идентификатор версии приложения (например, git-хэш или номер сборки), который SDK прикладывает к событиям и транзакциям. Позволяет увидеть, в каком релизе появилась проблема или произошла регрессия производительности.
