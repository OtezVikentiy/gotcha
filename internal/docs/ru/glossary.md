# Термины

**Проблема (issue)** — группа одинаковых событий (ошибок), сведённых по фингерпринту. У проблемы есть статус (не решено/решено/игнорируется) и ответственный.

**Событие (event)** — один конкретный случай: ошибка со стектрейсом, тегами, контекстом, данными пользователя и SDK.

**Фингерпринт (fingerprint)** — ключ группировки: по нему одинаковые ошибки попадают в одну проблему.

**DSN** — строка подключения проекта, которую вы указываете в SDK. Содержит публичный ключ и адрес приёма.

**Транзакция (transaction)** — единица производительности: один обработанный запрос/операция с длительностью. Состоит из спанов.

**Спан (span)** — вложенный отрезок внутри транзакции (запрос к БД, HTTP-вызов).

**Трейс (trace)** — сквозная цепочка связанных спанов/транзакций одного запроса.

**Web Vitals** — метрики скорости страниц браузера: LCP, INP, CLS, FCP, TTFB (p75).

**Apdex** — индекс удовлетворённости по времени отклика относительно порога T.

**Метрика (metric)** — числовой временной ряд (counter/gauge/histogram) с лейблами; агрегации p50/p95/avg/sum.

**Монитор (uptime monitor)** — периодическая проверка доступности (HTTP/TCP/DNS/heartbeat). При падении открывается **инцидент**.

**Оповещение (alert)** — правило + канал доставки: уведомление о новой проблеме, регрессии или всплеске.

**Регрессия (regression)** — статистически значимое ухудшение (латентности/Web Vitals) относительно базовой линии.

**Профиль (profile) / флеймграф** — снимок, куда уходит CPU/время выполнения по стеку вызовов.
