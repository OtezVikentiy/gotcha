# Проблемы

Раздел «Проблемы» показывает список ошибок, сгруппированных по фингерпринту, — по каждой видно уровень, тренд за 24 часа, число событий, статус и ответственного.

## Список и фильтры
Таблица сортируется по последнему событию, первому событию или частоте. Фильтры сужают список по статусу (не решено/решено/игнорируется), уровню (отладка…критическая), окружению, периоду и поисковому запросу по заголовку или culprit.

## Как работает группировка
Каждое входящее событие получает фингерпринт — ключ, вычисленный по типу ошибки, сообщению и месту в коде. Новые события с тем же фингерпринтом попадают в уже существующую проблему и увеличивают счётчик «Количество»; первый и новый фингерпринт создают новую проблему.

## Деталь проблемы
Сверху — стектрейс последнего события; ниже — список последних событий с возможностью переключиться на любое из них. Для каждого события доступны теги, данные пользователя, SDK и произвольные контексты, а также ссылка на связанный трейс в «Производительности».

## Действия
Проблему можно разрешить, отправить в игнор или вернуть в работу, а также назначить ответственного из участников проекта.
